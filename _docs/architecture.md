# Архитектура платформы КТ

## Публичные разделы сайта

### Главная страница
- Героический баннер (слоган, УТП, CTA: Программы + Подать заявку)
- Сводные KPI-счётчики (анимация при скролле): заёмщики, ссудный портфель, посевная площадь, поголовье
- Таб-переключатель: Прошлый год / Текущий год / Динамика
- Совокупные активы заёмщиков: земельный фонд, посевная площадь (зерновые и масличные отдельно), поголовье по видам, ссудный портфель
- Кредитный калькулятор: сумма/срок/ставка → аннуитет или дифф. → таблица платежей → CRM с параметрами
- Мини-карта ЗКО (SVG, статичная, клик → полная карта)
- Партнёры и аккредитация: КазАгро, Байтерек, ФРП, банки-партнёры
- Виджет событий: выезды, собрания, дедлайны сезонного кредитования
- Отзывы заёмщиков: текст + фото + хозяйство + район (из Directus CMS)
- Лента новостей: 3 последние карточки (из Directus)

### О компании
- История, миссия, направления деятельности
- Руководство и команда: фото, должность, контакт
- Блог / Слово председателя (из Directus)
- Награды, партнёрства, ассоциации
- Устав и нормативные документы

### Программы кредитования
- Сетка карточек: изображение, название, условия (ставка, срок, сумма, каникулы)
- Умный фильтр: тип деятельности (растениеводство/животноводство/смешанное), срок, сумма, залог
- Сравнение программ (side-by-side таблица, до 3 программ)
- Детальная страница: полное описание, требования к заёмщику, условия, документы, CTA «Подать заявку»
- Общие документы: инструкции, правила, шаблоны заявок, анкеты, согласия на обработку ПД
- Режим редактора (employee+): инлайн-кнопки «Редактировать / В архив / Дублировать / Создать» через Directus API

### Интерактивная карта ЗКО
- Движок: MapLibre GL JS (open-source) или Mapbox GL JS
- Данные: PostGIS → GeoJSON API (НЕ Grafana — она только для дашбордов)
- Боковая панель переключения слоёв

**Публичные слои (все пользователи):**
- Контуры районов ЗКО с названиями
- Метки хозяйств-заёмщиков (кластеризация при zoom-out), без финансовых данных
- Вид деятельности: цвет по типу (зерновые/масличные/животноводство/смешанное)
- Зона покрытия КТ
- Инфраструктура: элеваторы, ветпункты, ГСМ (OpenStreetMap)
- Бонитет почв (данные земельного кадастра РК)
- Виды угодий: пашня / пастбища / залежь
- Риски: засухи, паводки (открытые метеоданные)
- Сезонная активность: плановые даты сева/уборки по хозяйствам

**Слои только для сотрудников (employee+):**
- Тепловая карта ссудного портфеля в разрезе районов
- Статус займов: текущий / просроченный / закрытый (цветовое кодирование)

**Карточка хозяйства (popup при клике):**
- Публично: название, район, вид деятельности, площадь, поголовье
- Employee+: кнопка «Финансовая карточка заёмщика» → полное досье

### Наши проекты
- Карточки: текст, фотогалерея (lightbox), видео
- Цифры: объём финансирования, площадь, результат
- Хронология реализации
- Фильтрация: год / направление / район

### Новости
- Лента с датой, категорией, превью, полным текстом
- Категории: новости КТ / отраслевые / анонсы событий
- Управление: Directus CMS
- Подписка на уведомления

### Контакты
- Адреса, телефоны, email, режим работы
- Карточки сотрудников: фото, должность, прямой контакт
- Мини-карта офиса (Leaflet embed)
- Форма обратной связи → уведомление на корпоративную почту

### FAQ
- Поиск с мгновенной фильтрацией
- Аккордеон по категориям
- Нормативная база: ссылки на законодательство РК по кредитным товариществам
- Employee+: добавление/редактирование вопросов из интерфейса

## Приватные модули

### Личный кабинет заёмщика (borrower+)
- Дашборд: активный кредит, ближайший платёж, статус заявки, уведомления
- Документы: список из MinIO, скачивание по presigned URL, предпросмотр PDF
- График платежей: таблица (оплаченные / предстоящие / просроченные), прогресс-бар
- История операций
- Обращения (тикет-система): тема + текст + вложение → сотрудник отвечает в интерфейсе → уведомление заёмщику через Resend
- Типы обращений: досрочное погашение, реструктуризация, пролонгация, прочее
- Центр уведомлений: список, отметка о прочтении, фильтрация

### Аналитика (employee+)
- Grafana через iframe + SSO токен Keycloak (OIDC) — отдельный аккаунт не нужен
- Финансовые показатели КТ: портфель, динамика, просрочка
- Детализация по заёмщикам (из 1С через sync-svc): займы, графики, залоги, задолженности
- Статистика: заявки, обращения, посещения сайта
- Конструктор отчётности: период + метрики + выгрузка (Excel/PDF)
- Мониторинг залогов: срок страховки, дата инвентаризации, нарушения условий
- Предстоящие платежи: сводный список с фильтром и сортировкой
- Push-уведомления сотрудникам при контрольных событиях

### Экспертиза — BPM-модуль (employee+ / expert / admin)
- Отдельный Go-сервис (expertise-svc) + PostgreSQL + NATS JetStream

**FSM жизненного цикла заявки:**
```
Поступила → Первичный скоринг → Проверка безопасности → Экспертиза залогов
→ Юридическая проверка → Кредитный анализ → Кредитный комитет
→ Одобрена / Отклонена / Доработка → Формирование документации → Выдача кредита
```

**Карточка заявки — вкладки:**
1. **Общие данные** — заявитель, хозяйство, условия (формируется из карточки заёмщика)
2. **Финансовый анализ** — данные из 1С, кредитная история, платёжеспособность
3. **Залоговое обеспечение** — карточки залогов (самостоятельные сущности: высвобождение, привязка к новым кредитам, изменение)
4. **Документы** — файлы из MinIO, скачивание, аннотации
5. **Заключение** — чекбоксы рисков, текстовые поля, итог эксперта, загруженные/сформированные заключения
6. **История** — аудит-лог: кто/что/когда (только INSERT, никогда UPDATE/DELETE)

**Кредитный комитет (expert/admin):**
- Видят заявки только на этапе КК
- Просмотр всех заключений предыдущих этапов
- Голосование с отметкой (аналог подписи)
- Формирование протокола КК (PDF)

**Уведомления по событиям:**
- NATS событие при смене статуса
- Сотрудник следующего этапа: уведомление в интерфейсе + корпоративная почта
- Заявитель: Resend на личный email

**Формирование документации и выдача:**
- Прикрепление документов по замечаниям экспертизы
- Формирование договоров займа и графиков платежей
- Онлайн-подписание или прикрепление скан-файла
- Процедура выдачи → передача данных в 1С + запись в БД

### Почта и диск (employee+)
**Сотрудники:** Microsoft 365 или Google Workspace (SAML 2.0 → Keycloak SSO)
- Exchange/Gmail, SharePoint/Drive, Teams/Meet
- Ссылки встроены как deeplinks — самостоятельно не разрабатывается

**Заёмщики:** Resend (исходящие уведомления на личный email)
- Шаблоны: React Email
- Статус заявки, платежи, ответы на обращения, сброс пароля, подтверждение

### Виртуальный ассистент (все пользователи)
- Anthropic API (Claude) через assistant-svc (Go-прокси)
- System prompt: описание программ КТ, правила, контекст из Directus
- Rate limiting по IP
- Функции: FAQ, подбор программы, расчёт платежей, навигация по сайту
- Для авторизованных заёмщиков: помощь по ЛК
- При сложных вопросах: создание тикета → живой сотрудник

## Интеграции

| Система | Метод | Назначение |
|---------|-------|-----------|
| 1С | REST (HTTP-сервисы) + Go-брокер + Redis | Финансовые данные: займы, графики, залоги, задолженности. Cron каждые 20 мин, TTL кэша 30 мин |
| CRM/BPM внешняя | REST API + NATS | Приём заявок, передача статусов |
| Keycloak | OIDC / SAML 2.0 | Единая авторизация: сайт, Grafana, Directus, MinIO, M365/GW |
| Grafana | iframe + Keycloak SSO | Аналитика без отдельного входа |
| Resend | Go SDK | Транзакционная почта заёмщиков |
| M365 / Google Workspace | SAML 2.0 через Keycloak | Корпоративная почта+диск сотрудников |
| OpenStreetMap + кадастр РК | GeoJSON | Слои карты: инфраструктура, угодья, бонитет |
| Backblaze B2 | pg_dump + rclone cron | Бэкапы PostgreSQL (каждые 6 ч, хранение 30 дней) |
| Anthropic API | HTTPS | Виртуальный ассистент |

## Безопасность
- Шифрование at-rest: MinIO бакеты + PostgreSQL
- Шифрование in-transit: TLS через Cloudflare (Full Strict)
- RBAC: каждый эндпоинт — JWT middleware + роль
- Rate limiting: Redis (API форм заявок, ассистент)
- DDoS: Cloudflare
- Бэкапы: Backblaze B2, проверка восстановления
- Логи: Loki (централизованно)
- Аудит-лог Экспертизы: только INSERT

## Мобильное приложение
- React Native + Expo (НЕ WebView для основного приложения)
- WebView используется ТОЛЬКО для открытия внешней CRM-формы заявки
- Auth: OAuth 2.0 PKCE через expo-auth-session → Keycloak
- Токены: expo-secure-store (зашифровано)
- Push: Expo Notifications + FCM (Android) + APNs (iOS)
- Карта: react-native-maps (упрощённые слои)
- Офлайн: react-query + AsyncStorage (кэш графика платежей, документов)
- Сборка/деплой: EAS Build → App Store + Google Play
- OTA обновления (JS-часть): EAS Update
- Код переиспользуется из packages/shared: типы, API-клиент, утилиты
