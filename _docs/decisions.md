# Ключевые архитектурные решения (ADR)

Этот файл отвечает на вопрос «почему X, а не Y».
Не менять без явного обсуждения с владельцем проекта.

---

## Инфраструктура и деплой

**Oracle Cloud Always Free → k3s → Kubernetes**
- Oracle даёт 4 ARM vCPU + 24 GB RAM бесплатно (единственное конкурентное бесплатное предложение такого уровня)
- k3s — облегчённый Kubernetes, при росте мигрируем на полный k8s без переписывания манифестов
- Docker Compose — только для локальной разработки

**Cloudflare (бесплатно)**
- CDN + DDoS защита + SSL
- SSL режим: Full Strict (Cloudflare Origin Certificate на сервере)
- Не использовать Let's Encrypt — Cloudflare Origin Certificate надёжнее в связке с Cloudflare

## Бэкенд

**Go (Golang), а не Node.js / Python**
- Компилируемый, быстрый, минимальное потребление памяти
- Отличная конкурентность (goroutines) для обработки очередей NATS
- Микросервисы горизонтально масштабируются без проблем
- Фреймворк: Fiber или chi router (НЕ Echo, НЕ Gin — chi проще и надёжнее)

**NATS JetStream, а не Kafka / RabbitMQ**
- Kafka избыточна для нашего масштаба
- NATS JetStream проще в эксплуатации при сопоставимой надёжности
- Поддерживает: pub/sub, очереди, persistence, consumer groups

**gRPC для межсервисного транспорта**
- Строгая типизация через protobuf
- Эффективнее REST для внутренних вызовов
- REST только для внешнего API (/api/v1/...)

## База данных

**PostgreSQL + TimescaleDB + PostGIS (одна СУБД)**
- PostgreSQL: ACID для финансовых данных
- TimescaleDB: временные ряды (графики платежей, история показателей) — без смены СУБД
- PostGIS: геоданные карты (полигоны районов, координаты хозяйств)
- Одна СУБД закрывает три задачи — меньше операционной сложности

**Redis (Valkey)**
- Сессии, кэш API, rate limiting
- Кэш синхронизации с 1С (TTL 30 мин) — 1С не нагружается при каждом запросе

## Авторизация

**Keycloak, а не Auth0 / Clerk / собственная реализация**
- Enterprise-grade: RBAC + OIDC + SAML 2.0 + MFA из коробки
- SAML 2.0 критичен для федерации с Microsoft 365 / Google Workspace (SSO для сотрудников)
- PKCE flow для мобильного приложения
- Grafana, Directus, MinIO — все интегрируются по OIDC, отдельные аккаунты не нужны
- Self-hosted → данные не покидают инфраструктуру КТ

## Фронтенд

**Next.js 14+ App Router, а не Nuxt / SvelteKit / Remix**
- SSR/SSG из коробки (SEO критично для финансовой организации)
- React-экосистема переиспользуется в мобильном приложении (React Native)
- App Router — актуальный стандарт Next.js

**TailwindCSS**
- Утилитарный CSS: быстрая вёрстка, нет конфликтов стилей
- Поддерживается NativeWind (тот же подход в React Native)

**MapLibre GL JS (основной), Mapbox GL JS (резерв)**
- MapLibre — open-source форк Mapbox, без ограничений лицензии
- Mapbox — как резерв если нужны специфические тайлы
- НЕ Grafana для карты: Grafana не подходит для произвольных полигонов и ролевых слоёв

## Контент

**Directus (headless CMS), а не Strapi / Contentful / WordPress**
- REST + GraphQL API из коробки
- Ролевые права на уровне коллекций и полей (критично для нашей ролевой модели)
- Инлайн-редактор на сайте работает через Directus API — сотрудникам не нужна отдельная CMS-панель
- Self-hosted, интегрируется с Keycloak по OIDC

---

## Аналитика

**Grafana + Prometheus + Loki (kube-prometheus-stack)**
- Grafana: дашборды для раздела Аналитики + системный мониторинг сервисов
- Prometheus: метрики всех Go-сервисов
- Loki: централизованные логи всех подов
- Встраивается через iframe + Keycloak SSO — сотрудник не создаёт отдельный аккаунт
- НЕ использовать Grafana для карты — только для табличных дашбордов

## Почта

**Два разных контура — намеренно:**

| Контур | Инструмент | Назначение |
|--------|-----------|-----------|
| Транзакционная (заёмщики) | Resend + React Email | Исходящие уведомления системы: статусы заявок, платежи, ответы на обращения |
| Корпоративная (сотрудники) | M365 / Google Workspace | Полноценная корпоративная почта + диск + мессенджер |

**Resend, а не SendGrid / Mailgun**
- Создан для разработчиков, отличная доставляемость
- React Email шаблоны (тот же React что и на фронте)
- Бесплатный план достаточен для старта

**Brevo — только резервный SMTP** (не основной)

## Хранилище файлов

**MinIO (on-premise), а не AWS S3**
- Финансовые документы не должны покидать инфраструктуру КТ
- S3-совместимый API — при необходимости мигрируем на S3 без изменения кода
- Шифрование at-rest на уровне бакетов
- Интеграция с Keycloak по политикам доступа

**Backblaze B2 — только для бэкапов PostgreSQL**
- Самое дешёвое объектное хранилище ($0.006/GB)
- Используется исключительно для pg_dump бэкапов, не для рабочих файлов

## Мобильное приложение

**React Native + Expo, а не WebView / Flutter**
- WebView ОТКЛОНЁН: push-уведомления ограничены на iOS, Grafana iframe не работает, карта медленная, PDF плохо открываются — неприемлемо для финансовой системы
- Flutter ОТКЛОНЁН: Dart ≠ TypeScript, нет переиспользования кода с сайтом
- React Native + Expo: тот же React и TypeScript, packages/shared переиспользуется (типы, API-клиент, утилиты), нативные компоненты

**WebView используется ТОЛЬКО** для открытия внешней CRM-формы заявки (один экран, не архитектура)

**EAS Build + EAS Update**
- EAS Build: автоматическая CI/CD сборка → App Store + Google Play
- EAS Update: OTA обновления JS-части без прохождения модерации сторов

## Бэкапы

**pg_dump + rclone → Backblaze B2**
- Простота и надёжность: проверенный инструмент
- Расписание: каждые 6 часов, хранение 30 дней
- КРИТИЧНО: после настройки обязательно проверить полный цикл восстановления
- CronJob в Kubernetes (не внешний cron) — управляется вместе с кластером

## Мониторинг

**Двухуровневый:**
1. Grafana + Prometheus (внутренний) — метрики сервисов, алерты, логи
2. UptimeRobot (внешний, бесплатно) — пинг снаружи: если сервер упал, Grafana тоже молчит

**Uptime Kuma ОТКЛОНЁН** — дублирует Grafana+Prometheus, избыточно

## Синхронизация с 1С

**Cron каждые 20 минут (НЕ real-time)**
- 1С не предназначена для высокочастотных запросов
- Данные кэшируются в Redis (TTL 30 мин) + записываются в PostgreSQL
- При каждом запросе с сайта данные берутся из кэша, 1С не нагружается
- Real-time webhooks от 1С — рассмотреть в следующих итерациях если потребуется
